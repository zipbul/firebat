# firebat

TypeScript code-quality scanner. 15 detectors.

## Common principles — judgment basis

Every detector's judgment basis must be a **verifiable fact**: language/runtime specification (ECMAScript/TS semantics, AST node structure), binding/type identity closed by scope resolution, declared contracts (package.json fields, `node:`/`bun:` prefixes, tsconfig), and explicit user configuration (globs, allowlists).

**Guess-values are forbidden**: a guess-value is a judgment basis whose truth cannot be verified — a value whose role or meaning is inferred from names, filenames, paths, or notation conventions that consumers choose arbitrarily. The moment a guess is wrong it produces a false W (noise) or an unjust K (suppression), so guesses are never used as a judgment basis.

**Unclosable → hold**: when a judgment cannot be closed by facts, hold instead of producing a W (FN direction). Spec-defined names are treated as spec facts only after the target's identity is confirmed (binding resolution, type proof); name matching with unconfirmed identity may act only in the K direction (toward not producing a finding).

## waste detector

Reports variable declarations whose removal **changes neither the program's observable behavior nor the TS type-check result**.

**Observable behavior**: the kind, count, and order of side effects; function return values; reference identity exposed externally. Debugger inspection, stack-frame position, and readability through variable names are not included.

**Information-preservation exception (judged by form, distinct from readability)**: K only when the RHS has the *form* of a **bare literal** (magic number, bitmask, etc. — `0x3ffc`). Substituting the literal would leave only a meaningless magic value, making the name the sole carrier of information (prevents information loss). This is a *closed rule* judged on RHS node form, not a subjective readability call about "meaningful names" — if the RHS is an expression (member access, operation, cast, etc.: `req.user.id`, `u.role === 'admin'`), the expression carries its meaning through substitution, so a single-use binding is **always W** no matter how descriptive its name.

**Judgment procedure**: substitute the RHS expression at the use site (meaning-preserving syntactic adjustments allowed — parentheses, shorthand); if the behaviors above are preserved → W, otherwise K.

**Targets**: const/let/var variable declarations and destructuring bindings in every scope (module / function / block).

**Non-targets (other detectors' territory)**:
- Variables with zero uses (no-unused-vars territory)
- Exported bindings (require cross-module analysis — dependencies detector territory)
- Function parameters / class fields / enums / namespaces / import bindings
- Non-variable waste (unreachable code, dead branches, unused expression statements, unused imports)

**K examples (non-exhaustive)**: closure capture, type narrowing, snapshot before mutation, resource-handle lifetime, evaluation count/order preservation across multiple uses, await/yield position preservation, pinned reference identity (useEffect dependencies, etc.), context-sensitive expressions (this/super/private), naming of an opaque bare literal (information preservation — see above).

## error-flow detector

Reports structures where a thrown error or a Promise rejection fails to preserve one of **observability, propagation, or cause information** because of local syntax.

**Observability / propagation / cause information**: whether the thrown value / rejection reason reaches its intended handling point (caller, await, catch, logging) — observability and propagation; whether the original error's message/stack/cause remain traceable — cause. Style, readability, and notation conventions are not included.

**Judgment procedure**: if, under the current syntax, the thrown value / rejection reason reaches a reachable handling point and the original error's message/stack/cause are preserved → K; if any one is lost, swallowed, or cause-severed → W.

**Targets**: try/catch/finally, throw, Promise constructor/chain/await/return flows at function/file granularity. Judged on the OXC AST; type questions ("is this value a Promise") are supplemented with gildash semantic information.

**Non-targets (other detectors' territory)**:
- Cross-module rejection propagation, global unhandledRejection, caller-side non-handling (outside file scope)
- Async styles/conventions that preserve observability, propagation, and cause (lint territory)
- Meaningless constructs removable without behavior change (redundancy territory)

**K examples (non-exhaustive)**: explicit fire-and-forget (`void p`, `.catch` attached before discarding), returning the Promise to propagate to the caller, rethrowing the original after logging/transforming, preserving the original as `cause` on a new Error, intentional rejection absorption of auxiliary Promises under allSettled/race, normal cleanup inside finally (no throw/return).

## duplicates detector

Reports structures where a decision-bearing code unit (a declaration, or a run of consecutive statements inside a function) is replicated in two or more places whose **normal forms match under binding/type substitution alone** (literals are never substituted — see judgment procedure), breaking the **single change point** of one decision.

**Single change point**: whether one decision (logic, contract) has exactly one expression in code. If the same decision exists in several places as the same normal form, every copy must be changed together, and missing one produces an inconsistency bug. Accidental token/format similarity, author intent, and readability through variable names are not included.

**Structural-match exception (judged by role, distinct from intent)**: even with an equal normal form, K if the construct is a **decision-free skeleton** (pure delegation — a body that is nothing but a single parameter-unchanged call return; bodyless overload signatures; empty marker types; other protocol-forced structures). There is no decision to drift, so no co-change obligation arises. This is a *closed rule* judging a construct's role by form, not a subjective "was it copied with the same intent" call — if the construct carries a decision (control flow, computation, contract), an equal normal form is **always W**.

**Judgment procedure**: two constructs are W when their normal forms — with bindings (identifiers/parameters/type parameters declared inside the comparison unit), type annotations, and type arguments replaced by placeholders — match exactly and both carry decisions; K when the normal forms diverge or the construct is a decision-free skeleton. **Binding substitution is alpha-renaming — placeholders are numbered by first occurrence to preserve co-reference, not collapsed into one placeholder**: reuse of the same binding gets the same number, different bindings get different numbers. So `x+x` (`#0+#0`) and `x+y` (`#0+#1`) are different decisions, and `s.add(v); l.push(v)` (v reused) vs `s.add(k); l.push(i)` (different bindings) are different decisions — collapsing bindings into a single placeholder would false-merge them (FP). Numbers are assigned by deterministic traversal within the comparison unit (node / statement run / contract member set), so the rule is corpus-independent, binary, and closed. Free identifiers (references to declarations/imports outside the comparison unit), property names (including `#private` names — a `#name` selects which member slot is touched: a property-name role, never a substitutable value binding), and **literal values** are compared verbatim without substitution — calling a different target or using a different literal is a different decision. **Literals are content=decision everywhere and are never substituted** (all literals, not just rule-data or discriminant literals): "only one literal differs" near-duplicates cannot be separated by a closed rule into *constant drift of one decision* (e.g. tax rates `0.1`/`0.2`) vs *different decisions* (e.g. `x.kind==='A'` vs `'B'`, `withType('Logical')` vs `('Binary')`) — both are "exactly one literal differs" — so for zero FP they are **not detected** (literal variants unreported; see non-targets). Type-declaration bodies (interface/type) are decisions themselves and are not substituted.

**Targets**: two kinds of decision-bearing units. ① **Declarations** — functions/methods/classes, contract-bearing type/interface definitions, rule-bearing data declarations (mappings, lookup tables). Contracts compare structurally regardless of declaration syntax — the same member structure appearing as both an interface and a type alias is a duplicated contract. **Behavior-bearing named declarations (functions/methods/classes/data declarations) have no size floor** — a small duplicated function is still an addressable change point and is caught (prevents false negatives). ② **Statement runs** — replicated statement blocks inside function bodies. Judged on OXC AST normal forms.

**Decision-existence floor (anonymous inline expressions + contract type declarations)**: the decision-existence floor (`minSize`) applies not only to statement runs but also to **anonymous function expressions** (arrows / unnamed function expressions) and **contract type declarations** (interface / type alias). An anonymous inline lambda is inline code (the inline equivalent of a statement run), not a named change point; a contract type declaration is behavior-free pure structure, so a sub-floor micro-shape (`{line;column}` etc.) is universal vocabulary that independent modules converge on by accident — the interface equivalent of a trivial lambda (names are alpha-renamed away in normal forms, so naming adds no information; coincidence probability decays exponentially with structure size — node count as the information-content proxy). Below the floor a unit is too small to carry a driftable decision → K. A closed rule judged by form (node type + node count); matching itself remains exact normal-form equality (binary, closed) — filtering, at zero FP, the accidental isomorphisms of independent decisions: trivially identical lambdas (`(a,b)=>a-b` comparators, `s=>s.length>0` predicates, `n=>f(n,k)` projections) and micro universal-shape contracts (`interface P {line:number; column:number}`). **The cost is FN on genuinely copy-pasted sub-floor micro-contracts** — an intended trade in the same direction as literal-variant non-detection. **Behavior-bearing named declarations are exempt from this floor** (small ones are still change points).

**Closed rules for statement runs**: catch replicated statement blocks regardless of function boundaries, with the unit fixed by these rules.
- **Boundary**: only **consecutive sibling statements** within one BlockStatement.body. Never crosses branch/loop bodies; never starts or ends in the middle of a compound statement.
- **Minimum size**: normal-form AST node count at or above the threshold (`minSize`). Trivial runs below it (a single logging call, two assignments) are too small to carry a decision → K. This is a decision-existence floor, not a similarity threshold; matching is exact normal-form equality (binary, closed).
- **Extraction safety**: the run must be extractable as one function. If **2 or more** bindings declared inside the run are used outside it (live-out), or a return/break/continue transfers control across the run boundary → K (not extractable with a single return). live-out 0–1 and no control escape → W candidate.
- **Nesting**: within the same site set, only the **maximal run** is reported. A sub-run repeated across a wider site set is a separate clone.

**Non-targets (other detectors' territory)**:
- Repetition of a single literal/constant value (a constant-extraction problem, not a subtree clone — redundancy territory)
- Near-duplicates whose normal forms diverge through partial edits (near-miss / Type-3) and semantically equal but structurally different duplicates (Type-4): not closable without a similarity threshold — out of scope
- **Literal-variant duplicates** (identical structure, only literals differ): cannot be separated by a closed rule into "constant drift (W)" vs "different decisions (K)" → not detected (unsubstituted literals make the normal forms diverge, so they never match)
- Non-extractable runs (outward-used bindings, control escape) and sub-minimum runs (closed rules above)
- **Sub-floor anonymous inline expressions** (arrows / unnamed function expressions) and **contract type declarations** (interface / type alias): too small to carry a decision (accidental isomorphism of universal shapes) — behavior-bearing named declarations are exempt from the floor (closed rule above)
- Cross-module duplication of export surfaces (dependencies territory)

**K examples (non-exhaustive)**: externally-forced skeletons (pure delegation, overload signatures, framework registration shapes), similar code with diverging normal forms, data structures sharing shape but carrying no decision, sub-floor anonymous inline lambdas (trivial comparators/predicates/projections), sub-floor micro universal-shape contracts (`{line;column}` etc. — accidental convergence of independent modules, judged by the floor alone with no path/location exceptions).

## indirection detector

Reports function/type declarations that re-expose an existing target under a new name/layer **without adding any decision** (transformation-free), where inlining (removing) the layer **changes neither observable behavior nor the TS type-check result** (thin-wrapper, type-remap, interface-rewrap), or structures where such transformation-free delegation **accumulates past a depth threshold** (forward-chain, cross-file-forwarding-chain).

**Observable behavior**: identical to waste/error-flow — the kind/count/order of side effects, function return values, externally exposed reference identity, await/yield/throw positions, and type-check results. Debugger inspection, stack frames, readability through names, and domain naming are not included.

**Valuelessness exception (judged by form, distinct from readability)**: K when the delegation adds a decision or inlining would change behavior/types/identity/async contracts, in these forms — ① argument transformation (reordering, addition, omission, defaults, partial application, literal injection, rest↔identifier, destructuring unpacking, optional-chain calls), ② reaching a callback/identity position (reference/identity boundary), ③ pinned receiver (this/super/private/external object), ④ async/await/generator (→ error-flow), ⑤ return narrowing (type predicate / asserts), ⑥ visibility/protocol enforcement (decorator, override, overload, get/set, declaration merging, module augmentation, declare, .d.ts), ⑦ generic transformation (carries typeArg/typeParam). Each form closes on OXC AST node shape in the judgment procedure below (not name/method allowlists or meaning inference) — the same kind of closed rule as waste's information preservation and duplicates' skeleton exception, not a subjective "is this a meaningful abstraction" call. If the body is a parameter-unchanged single-call pass-through matching none of these forms, or the type is a pure synonym / empty rewrap without typeArg/typeParam, it is **always W** no matter how descriptive the name.

**Judgment procedure**: a single indirection layer (thin-wrapper / type-remap / interface-rewrap) is W if inline substitution preserves the behavior and type-check results above, K if it adds a decision or breaks preservation. A delegation chain (forward-chain / cross-file-forwarding-chain) is W when the chain of transformation-free delegation hops (each hop passing ① below) exceeds the depth threshold (accumulated depth decides — not inline removability).

- **Single function delegation (thin-wrapper)**: ① **body-form gate** — the function node is non-async (`async` absent) and non-generator; the body (a single ReturnStatement/ExpressionStatement for a BlockStatement, the expression for a concise arrow) is a single CallExpression whose arguments pass the parameters 1:1 unchanged in declaration order (non-rest positions: Identifier only — no SpreadElement; rest only as SpreadElement `...p`; no additions/omissions/reorderings/defaults/literal injection/optional-chain call `f?.()`; unpacking an ObjectPattern/ArrayPattern parameter's bindings as arguments is an object↔positional shape transformation → K — only unchanged passing of the parameter identifier itself is W); the callee is a free function identifier, or a member call whose object is a parameter of the delegating function (`p.m(...)`) and not ThisExpression/Super/PrivateIdentifier/a non-parameter external identifier; the body contains no AwaitExpression/YieldExpression; the return annotation is not TSTypePredicate/`asserts`; not a decorator / `override` / overload (a same-name bodyless signature declared alongside — the implementation included → K) / get-set accessor. ② **reference-identity gate** — scan the uses of the delegating function and, if it is a variable init, of every rebinding identifier followed in-file to a fixpoint: reaches outside direct-call position (arguments of CallExpression/NewExpression; operands of `===`/`!==`/`==`/`!=`/`instanceof`; init / assignment RHS / array element / SpreadElement / default; return / export / JSX attribute value) must be zero — a single one → K. If the delegating function is a **property init**, property aliasing does not close by form → conservative K. If the delegating function is exported and its uses live outside the file, ② cannot be proven in-file → out of thin-wrapper scope (cross-module below). ③ Passing ①+② → substitute the call site with the callee.
- **Type delegation (type-remap / interface-rewrap)**: only when there is no typeArg/typeParam/heritage `typeArguments`, replace the alias with the original type.

Single function delegation and type delegation close on file-local OXC AST form alone (no gildash semantic branches). Delegation chains trace transformation-free hops (each passing ①) over the import graph, deterministic over a fixed file set.

**Targets**:
1. **thin-wrapper**: a sync, non-generator **named function declaration or variable-init arrow/function expression** that passes the single-delegation gates (①+②). A valueless single layer whose inline removal preserves behavior and identity. (Class methods cannot close ② → excluded — see method delegation below.)
2. **forward-chain**: a chain of transformation-free delegation hops (each passing ①) within one file. The depth threshold is the judgment gate — only chains with `depth > maxForwardDepth` are produced; the chain equivalent of duplicates' `minSize` decision-existence floor (corpus-independent and deterministic under fixed option values). Accumulated depth decides, so per-hop reference-identity inline-safety (②) is not required.
3. **cross-file-forwarding-chain**: a forward-chain crossing file boundaries along the import graph. As with (2), each hop passes ① and the depth exceeds the threshold — a depth signal that asserts neither ② (in-file use proof) nor export-surface removability. Each hop's ① judgment closes independently in its defining file; only the chain linking uses the import graph. Cross-file circular delegation chains (import-graph cycles) are reported as cross-file-forwarding-chain regardless of the depth threshold (depth=-1 marker). Same-file cycles (a→b→a within one file) are call cycles = infinite recursion — a bug surfaced by type checking/runtime, not a static valueless layer → unreported (out of scope).
4. **type-remap**: a `type A = B` pure alias (no typeArg/typeParam). Any typeArg/typeParam is a generic transformation → **always K** (`type A = Readonly<B>` → K — structural equivalence is a semantic-layer judgment, unclosable, out of scope).
5. **interface-rewrap**: zero members, exactly one `extends` (zero = empty marker; multiple `extends A, B` is a composition decision → K), no `typeParameters`, no `typeArguments` on that heritage clause, not declaration merging (2+ same-name interfaces/classes in the same file), not module augmentation, not `declare`, and the file must be a **module file** (has top-level import/export declarations) — a script file (global scope, no top-level import/export) can merge same-name interfaces across files, which a single-file AST cannot close → always K. All closed on AST form.

**Non-targets (other detectors' territory)**:
- **async / await / generator delegation** → error-flow. `async`/`await` pass-throughs (including `return await f()`, `await f?.()`), `function*`/`async function*`, `yield`/`yield*` pass-throughs, `void f(x)` fire-and-forget, and un-awaited pass-throughs belong to async-contract / rejection-flow preservation.
- **Variable value aliases** → waste (non-exported `const a = b`). **Exported function/value rebindings** (`export const g = f`), `export *`, default/named re-export surfaces → barrel/dependencies. enum/namespace runtime-object aliases → waste/dependencies.
- **Cross-module thin-wrapper** — a single-delegation (thin-wrapper) candidate that is exported with uses outside the file, so the reference-identity gate (②) cannot be proven in-file. (Chains #2/#3 do not require ②, so they are unaffected.)
- **Empty class rewrap (`class A extends B {}`)** — a class creates a new runtime constructor value / `instanceof` / prototype identity, so inlining `A`→`B` breaks reference identity → **always K**. A closed rule distinct from the type-erased interface-rewrap (`IndirectionFindingKind` has no class kind).
- **get/set accessor delegation** — the call site is a property access, not a call, so the "substitute call site with callee" inline procedure does not close on form — permanently out of scope (K).
- **Class method delegation** — methods are reached through instances/prototypes (`c.m`, `C.prototype.m`), not variable bindings, so ②'s use-reach tracing cannot close in-file under instance aliasing → conservative K (isomorphic to property init). Single-delegation thin-wrapper targets only named function declarations and variable-init arrow/function expressions.

**K examples (non-exhaustive)**: callback/identity-position delegation (`xs.map(handle)`, `userHof(xs, inc)`, `el.addEventListener('click', handle)` pairs, `a === wrapper`, `new Worker(handle)`, `set.add(handle)`, `const cb = x=>f(x); xs.map(cb)`), default/partial-application/reordering/literal-injection wrappers, optional-chain call wrappers (`f?.(x)`), passing rest as an identifier or spreading a non-rest, destructuring-parameter unpack wrappers (`({a,b})=>f(a,b)`), member delegation on this/super/private/external-object/import-namespace receivers, class method delegation (② unclosable under instance aliasing), delegating functions in property inits (aliasing unclosable), `async`/`return await`/generator/`yield*` delegation, type-predicate/`asserts` return wrappers, decorator/override/overload/get-set/declaration-merging/`declare`/.d.ts forced structures, generic transformation aliases/rewraps (`interface Wrap<T> extends Base`, `type A = Readonly<B>`, `interface NumberSet extends Set<number>`), multi-extends composition and zero-extends empty markers, empty-extends interfaces in script files (cross-file merging possible), empty class rewraps (`class A extends B {}` — runtime identity), cross-module single-delegation thin-wrappers (export prevents in-file use proof), and every normal abstraction that adds a decision.

## dependencies detector

Reports structures in the whole import graph where **inter-module dependency contracts are unrealized or broken**.

**Inter-module dependency contract**: whether exports/files reach an actual consumption flow, internal references resolve to files, external packages' declarations and imports agree, import edges stay within allowed layers, and export surfaces avoid exposing the same origin twice. Code style, author intent, readability through names, and re-export syntax taste are not included.

**Whole-indexing premise (judgability precondition)**: judgments run over the whole program's import graph and consumption relations (imports, re-exports, type-references, calls), not a single file. A "zero consumers" verdict is true only when all consumers are in the graph, so partial scan scopes are not judged. Symbol-consumption judgments (dead-export, unused members) hold whenever the relation index is incomplete (FP prevention) — no W is built on missing relations. unused-file depends on graph completeness and entry-point resolution rather than relations. As waste/indirection close within a file, dependencies closes within a complete graph.

**Judgment procedure**: obtain the import graph, export symbols, consumption relations, cycle structure, and origin resolution from the whole program (the gildash semantic graph — not the OXC AST), then judge each kind binarily with closed rules over graphs/sets/globs (no similarity, no thresholds). Paths are normalized relative to the project root.

**Targets (10 kinds, 6 families)**:
1. **Reachability** — `dead-export`: an exported symbol with zero external consumers besides itself (a single consumer — test file or otherwise — means not dead; "test consumption isn't real use" is an unclosable filename guess and is never judged). A re-export chain that fails to reach a final consumer propagates deadness to the origin. Exports of files reported as `unused-file` are excluded from dead-export candidates (file-level reporting wins — no double reporting). **Exports of entry-point modules (package entry points ∪ user `entry` globs) are exempt from dead-export** — their consumers are by definition outside the graph (runners, string-loading tools, external packages): an external contract. When a module is consumed wholesale (namespace `*` import, `export *`, dynamic `import()` — the whole module-namespace object is received at runtime; gildash `meta.isDynamic` is the fact source), no per-export/member judgment is made for it. A side-effect-only import (`import './mod'`) contributes only file reachability and consumes no exports (not wholesale consumption). A **default export is consumed through the `default` slot** — consumer edges carry `default` while the symbol carries its local name plus `detail.isDefault` (gildash facts) — so a default export is live when its `default` slot (or its local name, for dual `export { x }; export { x as default }`) has a consumer; the dead-re-export propagation pass applies the same default-slot guard. **When the root package declares entry points but none resolve into the source graph** (a dist-pointing published library) **and the user declares no `entry`**, the public surface is unidentifiable — its real consumers are external — so **every dead-export and unused-member verdict is held**. `unused-file`: a file unreachable from entry points. **Opt-in** — reported only when the user declares `entry` globs (the declaration is the fact "the entry set is complete"). Undeclared → completeness unproven → hold (FN direction; entry points are never guessed from filenames — no explosion). Entry points = package entry points (package.json main/module/exports/bin — automatic facts) ∪ user-declared `entry` globs (facts). **No filename-convention seeds** (guess-values removed). Files matching user `ignore` globs are excluded from unused-file reporting. When reachability is computable (entry points exist), unreachable files' exports are excluded from dead-export (file-level reporting wins — no double reporting). `unused-enum-member` / `unused-ns-member`: at least one member call is attributed to the parent (enum/namespace) (parent-use evidence) while no call targets this member — attribution is judged by a closed rule: a call is attributed to the module when the call relation targets the parent's defining module, or the calling file imported the parent's name from that module and the call target has the `Parent.` prefix; any other `Parent.`-prefixed call → attribution unclosed → hold. Zero parent calls → hold. **Type-only members (`type`/`interface` inside a namespace) are excluded from candidacy** — they are consumed via type references and never appear in call relations, so the call oracle cannot observe their use (hold, FN direction).
2. **Cycles** — `circular-dependency`: two or more files forming a direct/indirect **runtime** import cycle. Judged on the **value-only dependency graph** — `imports` ∪ value `re-exports` relations (`import type` / `export type … from` edges are runtime-erased and excluded: a pure-type cycle, or a mixed loop whose runtime edges do not close, is not a runtime cycle) — with **one finding per non-trivial strongly-connected component** (a representative cycle, not one finding per elementary circuit). Single-file self-loops are not reported ("two or more files").
3. **Dependency declaration consistency** — `unused-dependency`: a package declared in dependencies/devDependencies but not imported in the TS import graph (peer/optional are host-provided contract declarations, never unused candidates). Consumption evidence = value imports ∪ **type-only imports** (`type-references` relations — `import type { X } from 'pkg'` is real consumption; if the type-references relation query degrades, every unused-dependency verdict at that root is held). Hold rules: ① every `@types/*` is held unconditionally (deadness does not close without merging tsconfig `types`/`typeRoots`, `extends`, `references`, and triple-slash references, and ambient globals are consumed with no import at all), ② **ambient-type packages resolved from declared facts are held** — tsconfig `compilerOptions.types` resolved via the TS compiler API (jsonc-safe, extends-merged, project references followed, sibling root `tsconfig*.json` as additive sources) plus each entry's owner package root, and `/// <reference types>` chains in project sources and in held packages' entry `.d.ts` (e.g. `@types/bun` → `bun-types`); when the ambient set is **unclosable** (unreadable extends/references, custom `typeRoots` without explicit `types`, an unparseable present tsconfig, an unresolvable declared `@types/*`, or a reference chain over bound) → hold every unused-dependency verdict at that root, ③ an installed `bin`-carrying package is held (script / git hook / `bunx` / manual invocation is outside the static graph, so non-use cannot be proven — the dep's package.json `bin` field is the fact source, covering binary-name≠package-name like `typescript`→`tsc`); a manifest that cannot be read (pnpm non-flat store, Yarn PnP, hoisting above rootAbs) is `unknown` → **hold** (absence of install-state is not W evidence), ④ a **declared dependency whose name is a Node builtin** (`buffer`, `events` — polyfill intent) counts as used when imported bare, because the target environment (Node core vs bundler polyfill) is not a readable fact. Only a confirmed no-bin, unimported, non-ambient library is reported. `unlisted-dependency`: an imported external package declared in none of dependencies/devDependencies/peerDependencies/optionalDependencies — judged on **value imports only** (a type-only import's contract may be satisfied by `@types/pkg`, which the unlisted check does not back-map, so type-only imports never manufacture an unlisted finding); bare Node builtin names (`module.builtinModules` is the runtime fact — Node resolves a bare core specifier to core regardless of any same-named package) and `node:`/`bun:`-prefixed specifiers are never unlisted.
4. **Resolution** — `unresolved-import`: an internal reference (import/re-export, type included) that does not resolve to a target file — held when the relative specifier resolves to a file that **exists on disk** (a non-TS asset outside the index, e.g. `./data.json` — outside the whole-indexing premise; applies to imports and re-exports alike).
5. **Re-export surface** — `duplicate-export`: same-named exports that are **independently defined** yet resolve to the same origin, exposed on two or more surfaces (surfaces exposed only through re-export syntax are not compared — re-export surface composition is barrel territory).
6. **Layer contract** — `layer-violation`: an import between two files matching different layer globs that is absent from the allowlist — fully inactive when layers are unconfigured.

**Non-targets (other detectors' territory / out of judgment)**:
- Partial scans / incomplete relation indexes — violate the whole-indexing premise → hold
- Re-export syntax form (`export *`, deep-import, barrel composition, and other styles) — barrel territory. dependencies sees re-exports only as consumption flow, not style
- Transformation-free delegation layers (thin-wrapper, forwarding-chain) — indirection territory. Delegation is just a graph edge
- Normal-form duplication of code bodies — duplicates territory. duplicate-export is about export surfaces, not implementation bodies
- Fan-in/fan-out/instability hotspot interpretation on top of the import graph — removed outright (magnitude thresholds are unclosable guess-values, not a fact-closable verdict); dependencies computes the raw graph (adjacency, cuts) and stops there — no hotspot layer exists anywhere in firebat
- Consumption in non-TS files (.js etc.) — firebat is TS-only, out of judgment (judged on the TS import graph only)
- Runtime reachability / dynamic loading — outside the static import graph
- Value references of enum/namespace members — uses not observable as calls are out of judgment (hold, FN direction)

**K examples (non-exhaustive)**: an export reaching a final consumer through re-exports; an export with at least one consumer (even a test-file-only consumer — filenames never split test/prod); a file reachable from user `entry` or package.json entry points; a file matching user `ignore` globs (not unused-file); unused-file held when the user declares no `entry`; per-export/member judgments skipped for modules wholly consumed via namespace `*` import / `export *`; a file reached only by a side-effect import (not unused-file); builtins (`node:`/`bun:` and bare `module.builtinModules` names), the self package, explicit ignore targets, **every `@types/*`** (deadness unclosable without tsconfig merging and ambient globals are consumed importlessly — total hold, FN), **ambient-type packages** declared via tsconfig `types` or reference chains (`bun-types`), **installed `bin`-carrying packages** (invocation outside the static graph is unprovable — the dep's package.json `bin` field is the fact source, `typescript`→`tsc` included), **manifest-unreadable deps** (pnpm non-flat / Yarn PnP / upward hoisting → `unknown` → hold — absent install-state is not W evidence), **declared builtin-name polyfills** (`buffer` — the target environment is not a readable fact), packages declared as peer/optional, external imports (not internal references), same-named exports pointing to different origins, same-named exports whose origin resolution failed (conservative non-merge), members held for unobserved parent calls or type-only kinds, unused-file with no entry points defined, dead-exports held under incomplete relations or an unresolvable public surface (a dist-pointing root), type-only cycles and single-file self-loops (not runtime cycles), a relative import to an existing non-TS file (outside the index → held), edges to files outside every layer.

## barrel detector

Under a **user-declared directory-surface policy**, reports structures where module consumption and export surfaces **deviate from the declared public-surface contract**. Declared ⇔ config `features.barrel` is `true`/object, OR the config key is ABSENT and the user explicitly selects the detector (`--only barrel`) — both are explicit user configuration, i.e. facts. **Explicit `false` always wins → fully inactive even under explicit selection** (the policy is a property of the codebase; a per-invocation flag does not overrule a durable negative declaration). **Fully inactive when the policy is not declared** — a convention becomes a judgment basis only when declared (same pattern as layer-violation).

**Module public-surface contract**: every directory is a nested module whose public surface is its single `index.ts` (a directory specifier resolving to `dir/index.ts` is a Node/TS resolution spec fact). The contract — ① **consumption (ImportDeclaration edges)** of a module outside one's own subtree goes only through that module's surface (a directory specifier); re-export edges are not consumption and are governed solely by ④, ② a surface body contains only explicit named re-exports (`export { … } from`, `export type { … } from`, and the single-name namespace forms `export * as ns from` / `export type * as ns from`), ③ re-exports are enumerable (named) everywhere — bare `export *` / `export type *` destroys the surface's enumerability (`* as ns` adds exactly one enumerable name → exempt, closed on the AST `exported` field), ④ every re-export's origin resolves inside the file's own directory subtree (own-subtree aggregation — including nested child internals — is surface composition, K), ⑤ a directory with outside consumption demand has a surface. Notation spelling (`./m` vs `./m/index` — the same module resolving to the same file; under NodeNext the explicit spelling is the only valid one), re-export reachability/duplication, and author intent/readability are not included.

**Nested module boundary (judged by form, distinct from intent)**: when the target directory is an ancestor of the importer file, the edge never leaves the importer's own enclosing module → K. This closes on path-prefix containment (segment-safe) — not an intent call ("it's an internal file, allow it"). Conversely, an ImportDeclaration descending into a nested child module's internal file bypasses that module's surface (and the surface-route remedy creates no cycle) → **always W**.

**Declaration premise (judgability precondition)**: judgments happen only under the declared policy and within the scan set. Specifier resolution failure → treated as external / held — absence of resolution is never a W basis (the resolver is not a full tsc reimplementation; tsconfig `paths` follow tsc's single-pattern selection — exact over longest-prefix wildcard, no fall-through). An `index.ts` that exists on disk but outside the scan set → held (absence from the scan set is not absence). `import type` / `export type … from` edges are judged identically as source-coupling facts (this is a source contract — runtime erasure is a runtime fact). Dynamic `import()` is invisible to this detector (hold). Files matching user `ignore` globs are excluded and create no demand (K-direction only).

**Judgment procedure**: resolve import/re-export specifiers to files, then judge binarily with closed rules over the importer↔target directory path-prefix relation + `index.ts` existence + AST statement form. No similarity, no thresholds.

**Targets (5 kinds, 4 families)**:
1. **Reach path** — `deep-import`: an outside-subtree **ImportDeclaration** (never a re-export edge) resolves to a non-index file in a target directory that **has** an `index.ts` (absent → owned by missing-index — no double reporting; target directory being an ancestor of the importer → K).
2. **Surface existence** — `missing-index`: a directory without `index.ts` into which at least one outside-subtree ImportDeclaration resolves (consumption demand = graph fact; demand attaches to the target file's immediate directory only). One finding per directory. No demand → hold — forcing a surface manufactures consumer-less exports (dead-exports).
3. **Surface form** — `export-star`: bare `export * from` / `export type * from` in any in-scope file (in index.ts this kind fires alone — no double reporting with invalid-index-statement; `export * as ns from` forms exempt per ③). Name-collision ambiguity is typecheck territory. `invalid-index-statement`: any index.ts statement that is not a conforming re-export form per ② (all imports included — named/type imports and side-effect imports, distinguished via evidence; bare `export *` is owned by export-star).
4. **Surface origin** — `cross-module-reexport`: in any file, a re-export (export-from / export-all / import-then-export / default re-export) whose origin resolves outside the file's own directory subtree; one finding per (statement, origin source). **No local-use exemption** (an intent guess — a single local reference would silently conceal a surface violation: an unjust K). Different contract clauses co-fire on one statement (e.g. foreign bare `export *` → export-star + cross-module-reexport), same-clause never twice.

**Non-targets (other detectors' territory / out of judgment)**:
- Policy not declared — fully inactive
- Explicit `…/index` spelling — a notation convention resolving to the same module (lint territory); under NodeNext, treating it as W would be a false W
- Unresolvable internal specifiers — dependencies (unresolved-import)
- `export *` name-collision ambiguity — typecheck (TS2308)
- Re-export reachability / dead / duplicate exports — dependencies
- Import cycles arising from complying with the policy — dependencies (circular-dependency)
- Transformation-free forwarding layers — indirection
- External packages / node_modules consumption — out of judgment

**K examples (non-exhaustive)**: same-directory consumption; consuming an ancestor-directory file (movement within the enclosing module); any consumption through a surface via a directory specifier (type-only included — the consumption itself satisfies the contract); named re-exports of own-subtree origins (child barrel aggregation); a directory with zero outside demand lacking an index (held); an index containing only `export type { … } from` (conforming); unresolvable specifiers (treated as external); files matching user ignore globs; `…/index`-spelled consumption (the same surface is traversed).

## giant-file detector

Reports source files whose **line count exceeds the effective line budget**. **Active by default** at the documented default budget `DEFAULT_MAX_LINES = 1000` — giant-file requires no configuration to run. Consumer config `features["giant-file"].maxLines` overrides the default with a non-negative integer; `true` / `{}` / absent → the default budget applies. **Explicit `false` always wins, even under `--only giant-file`** — a durable declaration of non-participation that a per-invocation flag cannot override. An invalid `maxLines` (negative, fractional, non-finite) is rejected by schema, not floored. Every finding discloses the effective budget: `metrics.maxLines`.

**Budget-exceedance (judged by count, distinct from design quality)**: the finding asserts one fact — `lineCount > budget` under the effective (configured or default) budget — and makes **no** claim that the file is oversized, incohesive, doing too much, or should be split (a large hand-written schema / lookup table / registry can exceed the budget and be perfectly sound; "is this file too big to be good" is unclosable, and the coupling detector was removed for asserting exactly such magnitude-as-defect verdicts). This magnitude threshold closes under a **two-prong criterion**: ① **pure-comparison verdict** — the finding is a comparison against a disclosed number and asserts no defect (vocabulary neutralization in the catalog/remedy text is load-bearing for this prong: with a defect narrative still attached, a default budget is exactly coupling's failure mode with better metrics); ② **coherent, satisfiable consumer policy with a complete lossless remedy set** — any file can be brought under a line budget without distortion: split it, exclude it (`exclude` glob), or re-budget it (raise `maxLines`). A fan-in budget fails prong ② — a shared hub cannot be brought under budget by any non-distorting change (only duplication, an anti-fix) — so neutralized coupling stays dead; this precedent does not readmit it, and future magnitude detectors must pass both prongs, not just prong ①. The kind name is a label for "over budget", not a quality verdict — catalog/remedy text carry no oversized/should-be-split VERDICT wording (naming "split/extract" as one remedy option among exclude/re-budget is fine; asserting the file needs it is not).

**Line-count rule (closed)**: `lineCount` = the number of ECMAScript line-terminator sequences in the raw source text (CRLF counted once; LF; CR; U+2028; U+2029), plus one iff the text does not end in a terminator; empty text = 0. Measured from source text directly (no AST — a parse-failed file is still counted, not held). A single long line (a minified or generated one-liner, however many characters) is 1 — an intentional FN (byte width is not a line count; exclude such files by glob).

**Judgment procedure**: within the scan set, `lineCount > maxLines` → W, else K. Binary integer comparison — no gradient, no severity (all files with `lineCount > maxLines` receive the same verdict, whether one line over or tenfold over); `maxLines = 0` flags every non-empty file. Two independent exclude escapes, both K-direction only: the **global scan-set `exclude`** (top-level config, drops a file from every detector — the file never reaches giant-file, or any other detector, at all) and the **detector-local `features["giant-file"].exclude`** glob array (giant-file-only — the file still reaches every other detector; only this detector's finding is suppressed). The detector-local form is the ecosystem test-exemption convention (eslint `overrides`-style line-budget exemption for `*.spec.ts`/`*.test.ts`) without paying the global-exclude cost of hiding the file from waste/duplicates/dependencies/etc. A file whose project-relative path (the same path the finding carries) matches any detector-local `exclude` glob produces no giant-file finding regardless of `lineCount`.

**Targets (1 kind, 1 family)**: `giant-file` — an in-scope `.ts` source file whose `lineCount` exceeds the effective line budget (configured `maxLines` or the documented default). One finding per file.

**Non-targets (other detectors' territory / out of judgment)**:
- A guess-value is an unverifiable INFERRED fact (a role or meaning inferred from names/paths/conventions); a documented default budget (`DEFAULT_MAX_LINES = 1000`) is a declared, disclosed policy parameter, not a guess — undeclared `maxLines` does not mean inactive, it means the default applies
- "Is this file cohesive / doing too much / should be split" — a design judgment with no fact anchor (out of judgment)
- Logical/semantic size, long functions, deep nesting — nesting/duplicates/indirection territory; giant-file measures physical lines at file granularity only
- `.d.ts` declaration files and non-`.ts` sources (`.tsx`/`.mts`/`.cts`) — outside the scan set
- Files matching the global scan-set `exclude` globs — never reach any detector, giant-file included
- Files matching the detector-local `features["giant-file"].exclude` globs — reach every other detector normally; only the giant-file finding is suppressed (e.g. exempting `**/*.spec.ts`/`**/*.test.ts` from the line budget while still running waste/duplicates/etc. on them)

**K examples (non-exhaustive)**: `false` declared (inactive, wins even under `--only`); a file at or under budget; a file matching the global scan-set `exclude` globs (generated code, data tables, large barrel surfaces — invisible to every detector); a file matching the detector-local `features["giant-file"].exclude` globs (test/spec files exempted from the line budget alone, per the ecosystem test-exemption convention); a single-line minified file of any width (counted as 1 line — K under any budget ≥ 1); a physically small but semantically complex file (a size fact says nothing about complexity).
