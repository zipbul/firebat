# PLAN — giant-file surgery (default budget + neutral verdict + closed counting)

## Ruling that reshapes the committed definition (550790e)

The committed definition made giant-file OPT-IN (no maxLines → inactive). The product owner overruled: **firebat is a consumer product — it must work with zero config.**

Doctrinal reconciliation — the TWO-PRONG criterion (tri-review hardened; "the comparison is always true" ALONE is vacuous — neutralized coupling would pass it too):
1. **Pure-comparison verdict**: the finding asserts only `lineCount > budget N (provenance disclosed)` — a claim that cannot be false. This prong REQUIRES the vocabulary neutralization (D6) in the same commit: with defect narrative ("too many responsibilities") still attached, a default budget is exactly coupling's failure mode with better metrics.
2. **Coherent, satisfiable consumer policy with a complete lossless remedy set** (the format/printWidth category firebat already ships): any file can be brought under a line budget without distortion — split it, exclude it by glob, or re-budget. A fan-in budget fails this prong: a shared logger at fanIn=50 CANNOT be brought under budget by any non-distorting change (only duplication — an anti-fix), so the only available reading of that finding is "defect", reopening the falsifiability gap. Neutralized coupling therefore stays dead; this precedent does NOT readmit it, and future magnitude detectors (nesting etc.) must pass BOTH prongs, not just prong 1.

A disclosed default is not a guess-value: a guess-value is an unverifiable INFERRED FACT (from names/paths/conventions); a documented default budget is a declared policy parameter inferred from nothing — the committed definition's "default = guess-value" clause is amended accordingly (honest amendment, not revisionism; the code already shipped `?? 1000` — this surgery legalizes and hardens it, it does not invent enablement).
