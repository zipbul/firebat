# 구현 (Build)

계획을 실제 코드로 변환하는 단계. TDD 기반.

## 절차

1. 계획 산출물을 `qa-engineer` 에이전트로 크로스 검증한다. "테스트 가능한 상태인가?"
2. `developer` 에이전트를 spawn하여 작업을 수행한다.
   - 통합 테스트를 먼저 작성한다.
   - 실행하여 RED를 확인한다 (AI가 항상 통과하는 테스트를 작성하는 실패 모드 방지).
   - 구현하여 GREEN을 확인한다.
   - 파일 수정마다 즉시 관련 테스트를 실행한다.
   - 실패 시 다음 파일로 넘어가지 않는다.
   - 복잡한 순수 로직에 한해 unit 테스트를 추가한다.
3. `code-reviewer` 에이전트를 spawn하여 산출물을 검증한다.
   - TypeScript strict 위반 없는가
   - 테스트가 의미 있는 시나리오를 커버하는가 (happy path만 아닌지)
   - 의존성 규칙 준수하는가
   - 불필요한 복잡성이 없는가
   - 테스트 컨벤션(AAA, 3파트 네이밍) 준수하는가
4. Critical/High 이슈가 있으면 수정 후 재리뷰한다.

## 게이트 (다음 단계 크로스 검증)

검증 단계로 넘어가기 전, `qa-engineer` 에이전트가 구현 산출물을 검토하여 "전체 검증으로 넘어가도 되는가?" 확인한다.

## 산출물

- 검증된 코드
- 통과하는 테스트
