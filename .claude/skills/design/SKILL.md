# 기획 (Design)

아이디어를 구체적인 요구사항과 범위로 변환하는 단계.

## 절차

1. 탐색 산출물이 있으면, `architect` 에이전트로 기획 진입 가능 여부를 크로스 검증한다.
2. `designer` 에이전트를 spawn하여 작업을 수행한다.
   - 사용자에게 질문하여 맥락과 의도를 파악한다.
   - 요구사항을 구조화하고 범위를 정의한다.
   - 2-3개 대안을 비교한다.
3. `design-reviewer` 에이전트를 spawn하여 산출물을 검증한다.
   - 요구사항이 구체적이고 측정 가능한가
   - 범위가 명확한가 (포함/미포함)
   - 모호한 표현이 없는가
   - 기존 아키텍처와 충돌하지 않는가
4. Critical/High 이슈가 있으면 보완 후 재리뷰한다.

## 게이트 (다음 단계 크로스 검증)

계획 단계로 넘어가기 전, `architect` 에이전트가 기획 산출물을 검토하여 "기술적 계획이 가능한 상태인가?" 확인한다.

## 산출물

- 합의된 요구사항
- 범위 정의 (포함/미포함)
- 선택된 접근법 + 근거
