# 검증 (Verify)

구현된 코드의 전체 품질을 검증하는 최종 게이트.

## 절차

1. 구현 산출물을 `release-engineer` 에이전트로 크로스 검증한다. "출시 가능한 상태인가?"
2. `qa-engineer` 에이전트를 spawn하여 작업을 수행한다.
   - `bun test` — 전체 테스트 실행
   - `bun run lint` — 린트 검사
   - `bun run format` — 포맷 검사
   - `bun run deps` — 아키텍처 의존성 규칙 검증
   - `bun run build` — 빌드 성공 확인
   - 엣지케이스 식별 및 누락된 테스트 보고
3. `verify-reviewer` 에이전트를 spawn하여 검증 결과를 리뷰한다.
   - 전체 테스트가 통과하는가
   - 누락된 엣지케이스가 있는가
   - 보안 이슈(OWASP)가 없는가
   - 빌드가 성공하는가
4. Critical/High 이슈가 있으면 수정 후 재검증한다.

## 게이트 (다음 단계 크로스 검증)

출시 단계로 넘어가기 전, `release-engineer` 에이전트가 검증 보고서를 검토하여 "출시 가능한가?" 확인한다.

## 산출물

- 전체 통과 보고서 (테스트, 린트, 포맷, 의존성, 빌드)
