# 계획 (Plan)

기획된 요구사항을 기술적 구현 계획으로 변환하는 단계.

## 절차

1. 기획 산출물을 `developer` 에이전트로 크로스 검증한다. "구현 가능한가?"
2. Plan Mode로 진입한다.
3. `architect` 에이전트를 spawn하여 작업을 수행한다.
   - 서브에이전트로 관련 파일/패턴을 조사한다.
   - 수정 파일 목록, 순서, 의존관계를 정의한다.
   - 기존 코드에서 재사용 가능한 유틸/패턴을 식별한다.
   - 테스트 전략을 수립한다 (어떤 테스트를 먼저 작성할지).
4. `plan-reviewer` 에이전트를 spawn하여 산출물을 검증한다.
   - 프로젝트 아키텍처 규칙(의존성 방향) 준수하는가
   - 수정 파일 목록이 완전한가
   - 기존 유틸/패턴을 재사용하는가
   - 테스트 전략이 타당한가
5. Critical/High 이슈가 있으면 보완 후 재리뷰한다.

## 게이트 (다음 단계 크로스 검증)

구현 단계로 넘어가기 전, `developer` 에이전트가 계획 산출물을 검토하여 "이 계획대로 구현 가능한가?" 확인한다.

## 산출물

- 승인된 구현 계획
- 수정 파일 목록 + 순서
- 테스트 전략
- 재사용할 기존 코드 참조
