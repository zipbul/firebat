# 탐색 (Explore)

"이거 가능해?", "어떤 방법이 있어?" 같은 탐색적 질문에 대응하는 단계.

## 절차

1. 사용자의 탐색 목적과 질문을 파악한다.
2. `researcher` 에이전트를 spawn하여 코드베이스, 웹, 문서를 조사한다.
3. `explore-reviewer` 에이전트를 spawn하여 조사 결과를 검증한다.
   - 조사 범위가 충분한가
   - 누락된 대안이 있는가
   - 근거가 신뢰할 수 있는가
4. Critical/High 이슈가 있으면 보완 후 재리뷰한다.

## 게이트 (다음 단계 크로스 검증)

기획 단계로 넘어가기 전, `designer` 에이전트가 탐색 산출물을 검토하여 "기획이 가능한 상태인가?" 확인한다.

## 산출물

- 조사 결과 요약
- 추천 방향 + 근거
- 대안 비교표 (해당 시)
